# 重要なルール
- コード修正は問題の本質を理解し、最終的にベストプラクティスに従うような形で修正するようにしてください。最小変更で修正しようとしてその場しのぎの場当たり的な対応は絶対に行ってはなりません。
- このプロジェクトは現在開発中でありユーザーはいません。変更時は後方互換を完全に捨てて新たな設計として適切な形で実装してください。後方互換のために無駄なコードが発生しないように注意してください。本番コード内にデッドコードや無駄に冗長な構成が残らないよう、常に注意を払ってください。
- エラー時にフォールバック処理をいれるときは、「本当にそのフォールバック処理が適切か」もう一度自身に問い直してください。
エラーが起きたときに落ちるのはアプリとして望ましい挙動です。エラーが起きたときに適切な処理をせずにそのまま動いてしまうのはアプリとして最悪の挙動です。そのフォールバック処理であれば、適切にエラーを処理してアプリ機能への影響がないと断言できるのか、よく考えたうえで実装してください。
